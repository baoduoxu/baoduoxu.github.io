Simple Parallel Models and Algorithm

Posted on 2024-01-25 Edited on 2024-02-27 In Parallel Computing Word count in article: 3.8k Reading time ≈ 3 mins.

Parallel Sum (Reduce)

给定 \(\{a_i\}_{i=1}^n\), 计算 \(\sum_{i=1}^na_i\).

这个问题作为引入并行计算的例子.

就是写成分治的算法来让子问题并行地执行, 最常见的套路.

Parallel Prefix Sum (Scan)

数组 \(a_0,\cdots,a_{n-1}\), 前缀和为 \(A_k=\sum_{i\le k}a_i=A_{k-1}+a_k\), 下文中所有的除法都默认下取整.

非并行的算法为 \(O(n)\).

根据 reduce 算法得到的树就可以直接通过搜索计算出每个前缀和, 这需要 \(O(n\log n)\) 的时间, 具体搜索规则如下:

  • 现在要计算 \(a_i\) 对应位置的前缀和, 从二叉树的根节点开始搜索 \(a_i\), 搜索路径是唯一的, 当路径向左时什么也不做; 当路径向右时, 则加上到达的结点的另一个兄弟节点.

并行思路 1

类似于计算序列和的 reduce 算法的其中一个, 减小问题规模:

  1. \(\mathrm{par\_for}_{0\le k\le n/2}(b_{k}=a_{2k}+a_{2k+1})\)

  2. 计算 \(\{b_k\}\) 的前缀和得到 \(\{B_k\}\), 且 \(A_0=a_0\).

  3. 然后 \[ \mathrm{par\_for}_{1\le k<n-1} \begin{cases} A_{k}=B_{k/2},&k\bmod 2=1\\ A_k=B_{k/2}+a_k,&k\bmod 2=0 \end{cases} \]

现在 work \(W(n)\) 和 span \(D(n)\) 分别满足

  • \(W(n)=O(n)+W(n/2)+O(n)=W(n/2)+O(n)\).
  • \(D(n)=O(\log n)+D(n/2)+O(\log n)=D(n/2)+O(\log n)\).

因此 \[ W(n)=W(n/2^k)+\sum_{0\le i\le k-1}O(n/2^{i})=O(n) \]

\[ D(n)=D(n/2^k)+\sum_{0\le i\le k-1}O(\log( n/2^{i}))=O(\log^2 n) \]

主方法应该更好用.

并行思路 2

分治可以得到更好的 span.

将 \(\{a_k\}\) 分成两部分 , 分别计算每一部分的前缀和得到 \(A^{(1)},A^{(2)}\), 这样可以得到前缀和 \(A\) 满足 \(A_k=A_k^{(1)},A_{k+|A^{(1)}|}=A^{(2)}_k+A_{|A^{(1)}|}^{(1)}\), 或者说, 这里的 \(A_{|A^{(1)}|}^{(1)}\) 应该写成 \(\sum_{1\le i\le n/2}a_i\), 这就意味着如果要使用分治, 就需要维护一个 offset, 每一个区间都有一个 offset.

更一般地, 设算法执行到某个阶段时下标区间为 \([s,t)\), 当前区间的 offset 为 \(f\), 则令 \(\mathrm{mid}=(s+t)/2\), 分区间 \([s,\mathrm{mid}),[\mathrm{mid},t)\), 区间 \([s,\mathrm{mid})\) 的 offset 为 \(f\), \([\mathrm{mid},t)\) 的 offset 为 \(f+\sum_{i\in[s,\mathrm{mid})}a_i\).

下面是一个例子, 对应的序列为 \(\{a_i\}_{1\le i\le 8},a_i=i\).

如果在并行时再计算求和 \(\sum_{i\in[s,\mathrm{mid})}a_i\) 显然开销较大, 则可以先预处理, 存到数组 \(L\) 中, 满足 \(L(s,\mathrm{mid})=\sum_{i\in[s,\mathrm{mid})}a_i\), 同时注意到分支过程中每一个出现的 \(\mathrm{mid}\) 都不会重复, 因此可以用一维数组 \(L_{\mathrm{mid}}\), 预处理 \(L\) 的过程可以通过计算序列和的 reduce 算法得到, 因此:

scan_up 预处理出数组 L 后, 用 scan_down 计算前缀和

对于 work 和 span, \[ W(n)=2W(n/2)+O(n),D(n)=D(n/2)+O(\log n) \] \(O(n)\) 的 work 和 \(O(\log n)\) 的 span.

还有一个不需要额外空间的改进版本:

并行前缀和的应用: filtering/packing

filtering 问题描述如下:

给定 \(\{a_k\in\mathbb{F}\}\) 和布尔函数 \(f:\mathbb{F}\to \{0,1\}\), 求满足 \(f(a_k)=1\) 的那些数, 并输出得到序列 \(\{b_k\}\), 保留这些数在 \(\{a_k\}\) 中的顺序.

并行地对序列 \(\{f(a_k)\}\) 求前缀和 \(p_k\), 以此获取 \(\{a_k\}\) 中每个满足 \(f\) 的数在 \(\{b_k\}\) 中的下标.

这需要 \(O(n)\) 的 work 和 \(O(\log n)\) 的 span.

Parallel Quick Sort

快排的基本思想是找一个 pivot \(p\), 把 序列中 \(\le p\) 和 \(>p\) 的分别放在 \(p\) 的左边和右边(这一操作称为 partition), 再分别对两者进行相同操作.

最直观的并行想法是 partition 之后对两部分并行地处理, 但是 partition 需要 \(O(n)\) 的时间, 因此 \[ W(n)=2W(n/2)+O(n),D(n)=D(n/2)+O(n) \] 得到 \(W(n)=O(n\log n),D(n)=O(n)\).

parallel partition

借助前面提到的 filtering, partition 的过程就可以看作是一个 filtering, 其中 \(f(x)=\mathbb{I}(x\le p)\). 那么这一个过程就可以用 parallel filtering 来实现.

Analysis of Quick Sort

非并行的快排的期望时间复杂度为 \(O(n\log n)\), 递归次数期望为 \(O(\log n)\).

设把第 \(i\) 个元素排序到最终位置花了 \(d_i\) 次 (在分治树的第 \(d_i\) 层), 那么算法整体的 work 实际上是 \(W(n)=\sum_{1\le i\le n}d_i\), span 为 \(D(n)=O(\log n)\times \max_{1\le i\le n}(d_i)\). 且 \(\mathbb{E}(d_i)=O(\log n)\).

称 \(g(n)\in O(f(n))\) 以很高的概率成立 (with high probability), 如果 \(\forall c\ge c_0\), \(\Pr(g(n)\in O(cf(n)))\ge 1-1/n^c\), 其中 \(c_0\) 是一个常数.

即 \(g(n)\in O(cf(n))\) 成立的概率至少是 \(1-1/n^c\).

设独立同分布的随机变量 \(X_1,\cdots,X_n\in\{0,1\}\) 且 \(P(X_i=1)\le p_i\), 令 \(X(n)=\sum_{i=1}^nX_i,f(n)=\sum_{i=1}^np_i\), 则 \(X(n)\in O(f(n)+\log n)\) w.h.p.

最终并行快排有 \(O(n\log n)\) 的 work 以及 \(O(\log^2n)\) 的 span w.h.p, 即 \(\Pr(D(n)\in O(\log^2n))\ge 1-1/n^c,\forall c>0\).

Chernoff bound \[ \Pr(X\ge (1+\delta)\mu)\le\exp\left(-\frac{\delta^2\mu}{2+\delta}\right) \]
