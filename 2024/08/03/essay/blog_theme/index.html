博客主题选择

我在大一暑假 (2022 年 8 月份) 搭建的这个博客, 在这之前我喜欢在知乎上分享内容, 但是感觉知乎上大神太多了, 有点不好意思把自己写的内容发在上面, 而且也不喜欢有太多的人看我的文章, 但是我希望有一种便捷的查看自己笔记以及将自己笔记分享给别人的方式, 我之前尝试过 Obsidian 移动端和语雀, 但都不是很满意(不过语雀用了一个学期, 上面还遗留了一些大物, 离散和电分的笔记), 后来了解到 Github Pages 与 Hexo, 搭建了这样一个博客.

由于有选择困难症, 主题的选择我花了好久, 最终选定了 Next, 因为它足够简洁, 但功能却十分完善(这依赖于有足够多的人用这个主题并去维护). 我不喜欢太花里胡哨的主题. 后来自己改了好久的字体, 英文字体换了好多衬线字, 从 Garamond 到 Palatino 再到现在的 CMU Sans Serif, 我发现我还真是很容易喜新厌旧. 中文字体就一直在用宋体. 代码字体之前在用 Cascadia Momo, 然后换成 Jetbrains Momo, 最后又换成了 Fira Code, 现在 Vscode 也是在用 Fira Code.

后来我发现我需要一个相对学术一点的主页, 但是大多数技术型博客的主页都是清一色的文章列表 (可能程序员喜欢这样), Next 就是这样, 我不喜欢文章列表这样喧宾夺主, 我希望点进我的网站先看到的是个人介绍. 但是 Hexo 没有太多这样的主题, 后来发现了 A4, 也就是现在在用的主题, 相对来说还是比较符合我的需求, 但确实功能不如 Next 完善, 比如修改字体以及代码块的一些细节设置, 但好在都是无关紧要的细枝末节.

与此同时, 我还具体看了另外两个框架, Hugo 和 Jekyll, 其实我看到好多人都在用 Jekyll Academic Pages, 虽然很符合我的需求, 但是就是不太想换, 可能只是不想换一个框架, 感觉怪麻烦的. 后来我在 Hugo 上找到了 Texify3, 类型的主题, 我个人是很喜欢 TeX 排版出来的文章的感觉的, 简洁严谨, 看起来很舒服, 但是最近折腾了一段时间, 一直有报错没搞定, 只能先搁置, 等保研事情完全结束之后再搞了. 可能要进行一次比较大的迁移.

以后可能会搞三个主页, 一个记录生活, 一个记录学习, 还有一个就单纯作为学术主页 (如果走学术路线的话), 感觉这样又好像太麻烦了, 但是觉得泾渭分明一些更好.
